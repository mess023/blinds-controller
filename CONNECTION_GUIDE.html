1 · Network Setup

🌐 Ethernet

Each frame runs on a wired Ethernet connection with a static IP address. Connect all four frames and your control PC to the same switch or router.

⚠️
Use wired Ethernet — no Wi-Fi Art-Net and OSC require low-latency, loss-free communication. Wi-Fi jitter will cause stuttering blinds. A gigabit switch costs less than the headache.

Configure your control PC to a static IP in the same subnet, e.g. 10.0.0.10:

2 · Art-Net Control

Art-Net UDP 6454

Art-Net is the primary control protocol and is compatible with any professional lighting console, media server, or software that supports DMX-over-Ethernet (MagicQ, ETC, Resolume, TouchDesigner, QLC+, etc.). All four frames live in Universe 1, addressed as 4-channel fixtures. A single broadcast packet drives all four simultaneously.

Universe & patch

Parameter Value
Protocol Art-Net
Transport UDP
Port 6454 (standard Art-Net port)
Universe 1 (Net 0 · Sub 0 · Universe 1 in most consoles)
Destination Unicast to each frame IP or broadcast 10.0.0.255
DMX packet length 512 bytes
ℹ️
Broadcast vs. unicast Sending to 10.0.0.255 (broadcast) delivers the same packet to all frames with one send. Use unicast (10.0.0.101 … .104) when you need per-frame override, or when the frames are on separate subnets.

DMX channel map

Each frame occupies 4 consecutive DMX channels. Every blind position is encoded as a 16-bit value split across two 8-bit channels (high byte first — MSB/LSB):

Position = (ch_MSB × 256) + ch_LSB  →  range 0 (fully retracted) to 65535 (fully extended).

⚠️

Patching examples

MagicQ / ETC / generic console: create a generic 16-channel fixture or patch 4 × 4-channel fixtures starting at addresses 1, 5, 9, 13 in Universe 1.

Resolume Avenue / Arena:

// Output → Add Art-Net → Universe 1 → destination 10.0.0.255 (broadcast)
// Map DMX faders: ch 1-2 = Frame 1 bottom, ch 3-4 = Frame 1 top, etc.

TouchDesigner:

# Art-Net OUT CHOP
network address  10.0.0.255   # broadcast, or individual IPs
port             6454
universe         0            # TD uses 0-indexed: universe 0 = Art-Net universe 1
# Output channels 1-16 map directly to the DMX channel table above

QLC+: Output → Add universe → Art-Net output → IP 10.0.0.255 port 6454 universe 1.

3 · OSC Control

OSC UDP 7000

OSC gives direct, human-readable per-frame control. It is ideal for scripted automation, TouchDesigner, QLab, or any software with an OSC output. Each frame listens on its own IP at port 7000. Arguments are floats in the range 0.0 – 1.0 (0.0 = fully retracted, 1.0 = fully extended).

Position control endpoints

OSC addressArgumentDescription
/btm/pos float 0.0 – 1.0 Move bottom blind to position (0.0 = retracted, 1.0 = extended)
/top/pos float 0.0 – 1.0 Move top blind to position

Example — move Frame 2 bottom blind to 75%:

Send UDP packet to  10.0.0.102:7000
OSC address:  /btm/pos
Argument:     0.75  (float32)

Calibration & utility endpoints

OSC addressArgumentsDescription
/calibrate none Calibrate both blinds on the target frame (measures full travel range)
/btm/calibrate none Calibrate bottom blind only
/top/calibrate none Calibrate top blind only
/status none Request a status reply (see below)

Status reply

When the frame receives /status, it replies back to your IP:port with a /status message containing 8 integer arguments:

Arg #NameValues
0calibrated_bottom0 = not calibrated, 1 = calibrated
1homed_bottom 0 = not homed, 1 = homed since power-on
2max_steps_bottom Total step count of full travel range
3pos_steps_bottom Current position in stepper steps
4calibrated_top 0 / 1
5homed_top 0 / 1
6max_steps_top Total step count of full travel range
7pos_steps_top Current position in stepper steps

TouchDesigner — OSC OUT CHOP example:

# OSC Out CHOP — Frame 1 bottom blind
network address  10.0.0.101
port             7000
address          /btm/pos
data type        Float
# Drive the channel value from 0.0 to 1.0 with a LFO or ramp CHOP

QLab — OSC cue example:

# Patch: target IP 10.0.0.101, port 7000
OSC address:  /top/pos
Arguments:    0.5
ℹ️
Tip — smooth movement via repeated OSC messages The motor firmware applies its own speed/acceleration limit, so you can jump straight to any target position. For cinematic slow motion, ramp the float value over time in your software (LFO, timeline, or tween).

4 · First-time Calibration

🔧 Required once after power-on

The stepper motors have no absolute encoder — the firmware must measure the full travel range before it can accept position commands. This only needs to be done once per power cycle; the calibration values are stored in flash memory.

Steps

  1. Power on the frame. Wait ~5 seconds for the ESP32 to boot and connect to the network.
  2. Send /calibrate (no arguments) via OSC to the frame's IP on port 7000.
  3. The blind will travel to the end-stop, then back to home position — this takes ~30 seconds.
  4. Once complete, position commands via Art-Net or OSC become active.
⚠️
Do not send position commands during calibration Any Art-Net or OSC position messages sent while calibration is running are silently ignored. Wait until both calibrated and homed flags in the /status reply are 1.

Web interface (optional)

Each frame also exposes a built-in web UI for manual control and diagnostics. Open a browser and navigate to the frame's IP address:

# Browser address bar

5 · Troubleshooting

SymptomLikely causeFix
Frame does not respond to Art-Net or OSC Calibration not done, or frame not reachable Ping the IP; send /calibrate via OSC; check switch LEDs
Blind moves to 0% or ignores commands DMX bytes out of range, or 16-bit split wrong Verify MSB/LSB order: ch N = high byte, ch N+1 = low byte
Only one blind moves per frame Incorrect DMX start address Check patch: Frame 1=ch1, Frame 2=ch5, Frame 3=ch9, Frame 4=ch13
Jerky or stuttering movement Packet loss / Wi-Fi being used Switch to wired Ethernet; check cable & switch
OSC status reply not received Firewall blocking inbound UDP 7000, or wrong source port Disable host firewall for the 10.0.0.0/24 subnet
Frame offline after power cycle DHCP lease or IP conflict Frames use static IPs — verify no other device has .101–.104
